# 拾谷 ShiGu｜前端

> 一套面向高级收纳需求的二次元资产管理系统前端实现

**拾谷 ShiGu** 是一个专业的二次元谷子（手办/周边）数字化资产管理平台，致力于提供"云展柜式"的资产检索体验与高效的收纳管理能力。

---

## 📋 项目概览

### 核心价值
- **数字化收纳**：将物理收藏转化为数字化资产，实现快速检索与精准管理
- **云展柜体验**：多维筛选 + 智能搜索，毫秒级定位目标谷子
- **数据导入**：支持从 Bangumi 批量导入 IP 作品与角色信息，提升录入效率
- **双端适配**：完美支持 PC 端与移动端，随时随地管理收藏

### 技术特色
- **现代化技术栈**：Vue 3 + TypeScript + Vite，确保开发效率与代码质量
- **类型安全**：完整的 TypeScript 类型定义，减少运行时错误
- **性能优化**：防抖搜索、懒加载、响应式设计，保证流畅体验
- **用户体验**：香槟镭射主题设计，精致的交互细节，友好的错误提示

### 关键能力
- ✅ **云展柜检索**：300ms 防抖搜索 + 多维筛选（IP/角色/品类/状态/位置）
- ✅ **位置树管理**：层级化位置管理，支持树形导航与详情展示
- ✅ **资产录入**：完整表单校验 + 图片上传 + IP-角色联动选择
- ✅ **IP与角色管理**：统一管理页面 + BGM 批量导入 + 关键词管理
- ✅ **品类管理**：独立的品类 CRUD 操作
- ✅ **快捷操作**：右键菜单、悬浮按钮，提升操作效率

### 设计规范
- **主题风格**：香槟镭射（Champagne Laser）主题
  - 主色：`#D4AF37`（香槟金）- 用于边框、图标高亮
  - 辅助色：`#F5F5F7`（明亮灰）- 用于卡片背景
  - 点缀色：`#A29BFE`（镭射紫）- 用于主要按钮、渐变效果
- **响应式设计**：PC 端表格展示，移动端卡片展示，自动适配
- **性能目标**：首屏加载位置树 + 谷子列表 ≤ 1.5s（内网环境）

## 🚀 快速开始

### 环境要求
- Node.js ≥ 20.19.0 或 ≥ 22.12.0
- pnpm ≥ 9（项目已锁定 packageManager: pnpm@9.0.0）

```bash
pnpm install          # 安装依赖
cp .env.example .env  # 参考示例创建环境变量
# 配置 VITE_API_BASE_URL=http://127.0.0.1:8000
pnpm dev              # 本地启动 http://localhost:5173

pnpm build            # 生产构建
pnpm preview          # 预览生产包
pnpm type-check       # 类型检查
pnpm lint             # ESLint
pnpm test:unit        # 单元测试（Vitest）
```

## 🔌 API 接口

### 谷子相关
- `GET /api/goods/` - 列表查询（支持分页、筛选、搜索）
- `GET /api/goods/{id}/` - 详情查询
- `POST /api/goods/` - 创建谷子
- `PUT /api/goods/{id}/` - 更新谷子
- `DELETE /api/goods/{id}/` - 删除谷子
- `POST /api/goods/{id}/upload-main-photo/` - 上传/更新主图

### 位置相关
- `GET /api/location/tree/` - 获取位置树
- `GET /api/location/nodes/` - 获取位置节点列表
- `GET /api/location/{id}/` - 获取位置详情
- `POST /api/location/nodes/` - 创建位置节点
- `PUT /api/location/nodes/{id}/` - 更新位置节点
- `DELETE /api/location/nodes/{id}/` - 删除位置节点

### 元数据相关
- **IP 作品**：`GET/POST/PUT/DELETE /api/ips/`
- **角色**：`GET/POST/PUT/DELETE /api/characters/`
  - `GET /api/ips/{id}/characters/` - 获取 IP 下的角色列表
- **品类**：`GET/POST/PUT/DELETE /api/categories/`

### BGM 导入相关
- `POST /api/bgm/search-characters/` - 搜索 Bangumi 作品并获取角色列表
- `POST /api/bgm/create-characters/` - 批量创建 IP 和角色（支持自动创建 IP）

> 具体以后端 `api.md` 为准，按需在 `src/api/*` 调整。

- 新增功能时注意移动端适配（使用响应式设计）

### 后端开发者

- 图片上传接口：`/api/goods/{id}/upload-main-photo/` 已对接
- 若限流阈值变化需同步前端（目前默认 60次/分钟）
- 注意字段变更时同步更新前端 `src/api/types.ts`

### 测试人员
- 覆盖核心流程：检索、筛选、上传、位置树增删改、IP/角色/品类管理、BGM 导入

- 移动端适配测试：不同屏幕尺寸下的交互体验
- 边界情况：空数据、大量数据、特殊字符输入、BGM 搜索结果为空
- BGM 导入流程：搜索、选择、导入、结果展示各步骤的测试
